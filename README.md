📊 Sales Analytics Dashboard (Power BI)

📌 Project Overview

This project focuses on analyzing sales data using Microsoft Power BI to gain insights into product performance, regional sales distribution, and overall revenue trends.

The dataset contains sales transactions of chocolate products sold by different salespersons across multiple countries. The objective of the project is to transform raw Excel data into meaningful visual insights through an interactive Power BI dashboard.

The dashboard helps stakeholders monitor key sales metrics, identify high-performing products, and understand regional sales performance.

🎯 Problem Statement

Organizations generate large volumes of sales data on a daily basis. However, without proper analysis and visualization, it becomes difficult to understand sales patterns and evaluate business performance.

The objective of this project is to analyze sales data and answer important business questions such as:

• Which products generate the highest sales revenue?

• Which regions contribute the most to overall sales?

• Which salespersons are performing the best?

• How do sales vary over time?

The goal is to convert raw sales data into a clear and interactive dashboard that supports data-driven decision-making.

📂 Dataset

The dataset used in this project is stored in Microsoft Excel (.xls) format and contains sales transaction records.

Dataset Fields

• Sales Person – Name of the salesperson responsible for the sale

• Geography – Country or region where the sale occurred

• Product – Name of the chocolate product sold

• Date – Date of the sales transaction

• Sales – Total sales value generated

• Boxes – Number of product boxes sold

This dataset allows analysis of sales performance across different products, regions, and salespersons.

🛠 Tools Used

• Microsoft Power BI – Data cleaning, modeling, and dashboard visualization

• Microsoft Excel (.xls) – Source dataset used for analysis

🔄 Project Workflow

The project was completed using the following steps:

1. Data Collection

• The dataset was obtained in Excel format containing sales records of chocolate products.

2. Data Import

• The Excel dataset was imported into Power BI Desktop.

3. Data Cleaning

Basic data preparation was performed in Power BI including:

• Verifying column data types

• Removing unnecessary rows

• Preparing the dataset for analysis

4. Data Modeling

• The dataset was structured properly to support accurate analysis and visualization.

5. Dashboard Development

• Various visual elements were created in Power BI to present the data effectively.

6. Insight Generation

• The dashboard was used to analyze sales trends and identify key business insights.

📊 Key Performance Indicators (KPIs)

The dashboard tracks several key metrics to evaluate sales performance:

• Total Sales Revenue – Total revenue generated from product sales.

• Total Boxes Sold – Total number of boxes sold across all transactions.

• Total Transactions – Number of sales records in the dataset.

• Average Sales per Transaction – Average revenue generated per transaction.

• Sales by Product – Revenue contribution of each product.

• Sales by Geography – Sales distribution across regions.

• Sales by Salesperson – Performance analysis of individual salespersons.

📈 Dashboard Features

The Power BI dashboard includes several interactive visualizations such as:

• KPI cards showing total sales and boxes sold

• Bar charts displaying sales by product

• Regional sales comparison by geography

• Salesperson performance analysis

• Time-based sales trends

• Interactive slicers for filtering data

These features allow users to explore the dataset and gain deeper insights into sales performance.
